# ホロライブ推し探しチャート

質問に答えていくと、あなたに合ったホロライブメンバー（推し）を提案する静的Webアプリ。

## 特徴

- サーバー不要。HTML/CSS/JS のみの完全静的サイト
- 対象: hololive JP / EN / ID（トップページで複数選択可）
- 卒業生を含めるかどうかも選択可能
- 診断モード2種: サクッと10問 / じっくり30問（`data/questions.json` の `modes` で定義）
- パラメータ（10段階×9軸）＆タグのマッチングで推し候補トップ3を提示
- 結果にはライバーのパラメータとあなたの嗜好を重ねたレーダーチャートを表示
- ヘッダーからメンバー一覧（ブランチ・卒業生フィルタ付き）を閲覧可能

## 構成

```
index.html          エントリポイント
css/style.css       スタイル
js/app.js           アプリ本体（UI・スコアリング）
data/members.json   メンバーDB（パラメータ・タグ付き）
data/questions.json 質問と回答の重み定義
```

### データ分離について

メンバー・質問データは `data/*.json` に完全分離してある。
将来サーバーでデータ管理する場合は `js/app.js` の `DataSource.load()` の
fetch 先を API エンドポイントに差し替えるだけでよい。

### スコアリング（マッチ度は絶対値）

- 各回答がパラメータ重み（energy / healing / talk / game / music / comedy / chaos / cute / cool、メンバー側は10段階）とタグを加算
- ユーザー嗜好は回答重みの最大軸を10として正規化（レーダーチャートの破線と同一のベクトル）
- マッチ度 = パラメータ一致（9軸の平均差が小さいほど高い、最大90点）+ タグ一致ボーナス（最大10点）
- 相対値ではないため、レーダーチャートの重なり具合とマッチ度%が連動する。100%は形がほぼ完全一致した場合のみ

## ローカルで動かす

`fetch()` を使うため、file:// では動かない。簡易サーバー経由で開く:

```sh
npx serve .
# または
python3 -m http.server 8000
```

## GitHub Pages で公開

1. GitHub にリポジトリを作成して push
2. Settings → Pages → Source を「Deploy from a branch」、Branch を `main` / `(root)` に設定
3. `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される

ビルド不要。そのまま動く。

## データ更新

- メンバーの追加・卒業: `data/members.json` の `members` を編集（`status`: `active` / `graduated`）
- 質問の追加・調整: `data/questions.json` を編集（質問数は自動反映）

### メンバーのフィールド

| フィールド | 内容 |
|---|---|
| `color` | 推し色（カード装飾に使用） |
| `emoji` | アバター絵文字 |
| `description` | 特徴・紹介文（結果カードとメンバー詳細モーダルに表示） |
| `twitter` | X(Twitter) ハンドル（公式タレントページから取得） |
| `youtube.channelId` | YouTube チャンネルID（公式タレントページから取得した実ID） |

`youtube.channelId` があるメンバーは、結果画面で公式 YouTube 埋め込み
（アップロード動画プレイリスト `UU…`）を表示できる。無い場合は検索リンクにフォールバック。

## 画像について

カバー社の二次創作ガイドライン上、公式画像の転載は不可のため画像は使わない。
代わりに推し色 + 絵文字アバターと、YouTube 公式埋め込み（規約上OK）を使用。

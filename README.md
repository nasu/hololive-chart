# 推しあわせ（ホロライブ / にじさんじ / IRIAM 推し探しチャート）

質問に答えていくと、あなたに合ったVライバー（推し）を提案する静的Webアプリ。
トップでブランド（箱）を選んでから診断に入る。

## 特徴

- サーバー不要。HTML/CSS/JS のみの完全静的サイト
- **マルチブランド対応**: ホロライブ（74名収録）/ にじさんじ（209名収録）/ IRIAM（タイプ診断）
  - ホロライブ: JP / EN / ID、卒業生の含否選択
  - にじさんじ: JP / EN、卒業生の含否選択、**女性/男性ライバーの絞り込み**
  - IRIAM: 個人ライバー数万人規模で全員収録が不可能なため、**12のライバータイプ診断**方式。
    結果にアプリ内検索タグと大手事務所の入口を提示（タイプ×事務所の対応付けは根拠データがないため行わない）
- 診断モード2種: サクッと10問 / じっくり30問
- パラメータ（10段階×9軸）＆タグのマッチングで推し候補トップ3を提示
- 結果にはライバーのパラメータとあなたの嗜好を重ねたレーダーチャートを表示
- ヘッダーからメンバー一覧（フィルタ・詳細モーダル付き）を閲覧可能

## 構成

```
index.html                    エントリポイント
css/style.css                 スタイル
js/app.js                     アプリ本体（UI・スコアリング、ブランド非依存）
data/brands.json              ブランド一覧（ランディングに表示）
data/hololive/members.json    ホロライブ メンバーDB
data/hololive/questions.json  ホロライブ 質問定義
data/nijisanji/members.json   にじさんじ メンバーDB
data/nijisanji/questions.json にじさんじ 質問定義
```

### データ駆動設計

文言・ブランチ構成・性別フィルタの有無はすべて `members.json` の `brand` メタと
メンバーデータから自動決定される。アプリ本体（js/app.js）はブランドを知らない。

- 新ブランド追加 = `data/<id>/` に2ファイル置いて `brands.json` に1行足すだけ
- 性別フィルタはメンバーに `gender`（f/m/u）が2種以上あるブランドのみ自動表示
- 将来サーバー管理に移行する場合は `DataSource` の fetch 先を差し替えるだけ

### メンバーのフィールド

| フィールド | 内容 |
|---|---|
| `params` | 9軸パラメータ（10段階）: energy/healing/talk/game/music/comedy/chaos/cute/cool |
| `tags` | マッチング用タグ |
| `gender` | f / m / u（未設定なら常に表示対象） |
| `color` / `emoji` | 推し色・アバター絵文字（公式画像は権利上使用しない） |
| `description` | 特徴・紹介文（自前の文章。公式文の転載はしない） |
| `twitter` | X ハンドル（公式サイトから抽出） |
| `youtube.channelId` | YouTube チャンネルID（公式サイトから抽出した実ID） |

### スコアリング（マッチ度は絶対値）

- ユーザー嗜好は回答重みの最大軸を10として正規化（レーダーチャートの破線と同一ベクトル）
- マッチ度 = パラメータ一致（9軸の平均差、最大90点）+ タグ一致ボーナス（最大10点）
- レーダーチャートの重なり具合とマッチ度%が連動する

## 画像・権利について

カバー社・ANYCOLOR社とも二次創作ガイドライン上、公式画像の転載は不可のため画像は使わない。
代わりに推し色+絵文字アバターと、YouTube公式埋め込み（規約上OK）を使用。
説明文も公式プロフィールの転載を避け、すべて自前の文章。

## ローカルで動かす

```sh
npx serve .
# または
python3 -m http.server 8000
```

## デプロイ

`main` に push すると GitHub Actions（.github/workflows/deploy.yml）が
GitHub Pages に自動デプロイする。

## データ更新

- メンバーの追加・卒業: 各 `members.json` の `members` を編集（`status`: `active` / `graduated`）
- 質問の追加・調整: 各 `questions.json` を編集（`modes` で10問/30問の構成を管理）
